# Data Mining Final Project: Smartphone Price Range Prediction and Market Segmentation

Course: Data Mining / Academic Project  
Type: Data Analytics and Predictive Modeling Project  
Role: Data Analysis, Modeling, and Presentation Contributor

## The Problem

Smartphone prices often vary even when devices appear to have similar specifications. This project was developed to analyze how smartphone features influence pricing and to identify meaningful market segments based on technical specifications. The goal is to help users better understand whether a smartphone's price is reasonable based on its hardware and design attributes.

## Technical Stack

- **Data Mining Tool:** RapidMiner
- **Analysis:** Classification, Clustering, Association Rule Mining
- **Documentation:** Research Paper, Presentation Deck, Project Video
- **Other Tools:** GitHub

## System Architecture & Logic

### 1. Decision Tree Classification
A Decision Tree model was used to classify smartphones into price levels based on technical specifications. This helps identify which features most strongly influence whether a device belongs to a low, medium, or high price range.

### 2. k-Means Clustering
k-Means clustering was applied to group smartphones into natural segments based on shared characteristics. This technique helps reveal patterns in the market and distinguish groups such as budget, mid-range, and flagship devices.

### 3. Association Rule Mining
Association Rules were used to discover combinations of smartphone features that frequently appear together. This provides insight into which specifications are commonly linked to certain pricing patterns.

### 4. Market Insight Generation
The project combines predictive and descriptive data mining methods to better understand smartphone pricing behavior and market structure.

### 5. Data-Driven Consumer Perspective
The study was designed to support more informed decision-making by showing how technical specifications relate to perceived value and market positioning.
